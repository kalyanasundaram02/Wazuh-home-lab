# Wazuh Server Setup

## Host OS

Ubuntu Server 24.04 LTS, static IP `192.168.0.100`.

> Originally attempted on Ubuntu Server 26.04, but a networking issue (VS Code Remote-SSH stalling over a Bridged adapter) led to standardizing on 24.04 LTS instead, to match the version used in the institute's own lab environment. Full details in [Troubleshooting Log](07-troubleshooting-log.md).

## What was installed

The full Wazuh stack was installed on the single Ubuntu Server host:
- Wazuh manager
- Wazuh indexer
- Wazuh dashboard

Additionally, Suricata was installed on the same host as a network IDS, with its `eve.json` output ingested by Wazuh as a log source (see [Suricata IDS Integration](6.suricata-ids-integration.md)).

## Key configuration

Full file: [`configs/manager-ossec-config.xml`](../configs/manager-ossec-config.xml)

A few sections worth calling out:

**Local log sources monitored** — this is what makes host-based detection possible:
```xml
<localfile>
  <log_format>syslog</log_format>
  <location>/var/log/auth.log</location>
</localfile>

<localfile>
  <log_format>json</log_format>
  <location>/var/log/suricata/eve.json</location>
</localfile>
```
The first block is *why* SSH login failures/successes generate alerts at all — without it, Wazuh has no visibility into `auth.log`. The second block is the bridge between Suricata and Wazuh: Suricata writes its own alerts to `eve.json`, and this line tells Wazuh to read and correlate them into its own alert pipeline.

**Remote agent listener** (not actively used in this exercise, since the target is the manager itself, but configured for future agent-based hosts):
```xml
<remote>
  <connection>secure</connection>
  <port>1514</port>
  <protocol>tcp</protocol>
</remote>
```

**Alert logging** — `logall` and `logall_json` are enabled, meaning every event (not just ones that match a rule) is archived. Useful for a lab where you want to go back and inspect raw traffic later, though in a production SOC this is usually dialed back for storage/performance reasons.

**Indexer connection** — the manager forwards alerts to the local Wazuh indexer over `https://127.0.0.1:9200` using mutual TLS certs (`filebeat` certs), which is what powers the Wazuh dashboard queries used throughout this project.

## Static IP

Both Kali (`192.168.0.7`) and the Ubuntu Server (`192.168.0.100`) were configured with static IPs to keep addressing consistent across reboots — important in a home lab where DHCP lease changes would otherwise break agent enrollment and make alert correlation confusing over time.
